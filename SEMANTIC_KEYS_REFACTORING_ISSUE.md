# Рефакторинг семантичних ключів та статус перекладів

## Опис проблеми
У проекті триває перехід від використання генеричних ключів з цифровими суфіксами (наприклад, `patient_1`, `patient_2`, `bear_1`) до семантичних ключів (наприклад, `patient_medical`, `patient_enduring`, `bear_animal`, `bear_endure`).

Це необхідно для:
1.  **Уникнення неоднозначності**: Різні значення одного й того самого англійського слова вимагають різних перекладів на українську, кримськотатарську, нідерландську та німецьку мови.
2.  **Покращення якості перекладу**: Семантичні суфікси дають контекст перекладачам (або ШІ-моделям).

## Поточний стан (на 1 лютого 2026 р.)
Проведено масове оновлення ключів у файлах:
- `src/lib/data/words/levels/*.json` (списки слів)
- `src/lib/data/transcriptions/levels/*.json` (транскрипції)
- `src/lib/data/translations/*/levels/*.json` (переклади для всіх мов)

### Виявлені аномалії
Під час аналізу `git diff HEAD` було виявлено наступне:

1.  **Англійські заглушки**: Багато перекладів для української (`uk`), кримськотатарської (`crh`), нідерландської (`nl`) та німецької (`de`) мов все ще містять англійські слова замість перекладу (наприклад, `"patient_2": "patient"`).
2.  **Некоректні автоматичні переклади**:
    - У нідерландській мові (`nl`): `we_pronoun` перекладено як `weekend`, `later` як повідомлення про помилку, `agent` як `Gevolmachtigde`.
    - У німецькій мові (`de`): `hand` перекладено як `anziehen`, `arm` як `Schwenkarm`.
3.  **Частковий рефакторинг**: Деякі ключі були автоматично змінені, але значення залишилися застарілими або некоректними.

## Що потрібно зробити в наступних сесіях
1.  **Валідація перекладів**: Перевірити всі нові семантичні ключі на наявність коректних перекладів у всіх мовних файлах.
2.  **Заміна заглушок**: Знайти всі випадки, де значення перекладу збігається з англійським оригіналом (якщо це не запозичене слово), і замінити їх на правильний переклад.
3.  **Синхронізація мов**: Переконатися, що набір ключів ідентичний у всіх мовних файлах для кожного рівня.
4.  **Оновлення скриптів**: Перевірити скрипти в `scripts/maintenance/`, щоб вони підтримували нову семантичну структуру ключів.

## Корисні команди для перевірки
- `node scripts/analysis/analyze_all_translations.js` — для виявлення відсутніх або некоректних перекладів.
- `git diff HEAD` — для перегляду останніх змін у ключах.
